Ответы на вопросы:
● Что такое k8s? 
 это открытое программное обеспечение для автоматизации развёртывания, масштабирования и управления контейнеризированными приложениями/
 
● В чём преимущество контейнеризации над виртуализацией? 
Rонтейнеризация легче, быстрее и дешевле, чем классическая виртуализация.
Меньший вес сущности контейнера, контейнер легко импортируемый, образ контейнера легко перенести/

● В чём состоит принцип самоконтроля k8s? 
Принцип самоконтроля Kubernetes (self-healing) — это способность кластера самостоятельно поддерживать заданное состояние, без ручного вмешательства.
K8S перезапускает отказавшие контейнеры, заменяет и завершает работу контейнеров, которые не проходят определённую пользователем проверку работоспособности.

● Как вы думаете, зачем Вам понимать принципы деплоя в k8s? 
Понимание принципов деплоя в k8s нужно, чтобы не бояться инфраструктуры,не зависеть на 100% от DevOps, быстрее находить причину проблем, говорить с командой на одном языке.

● Какие типы нод есть в k8s, каковы их базовые функции?
Control Plane Node (master-узел) - хранит и поддерживает желаемое состояние кластера, принимает все команды (kubectl), принимает решения что, где и когда запускать, следит за здоровьем кластера
Worker Node (рабочая нода) - запуск контейнеров, контроль состояния Pod’ов, передача статуса в control plane, сетевое взаимодействие и storage
Специализированные worker-ноды (логическое разделение) - технически это worker nodes, но с особыми ролями(Infra nodes - для ingress controller,monitoring (Prometheus), logging (ELK),
Compute / GPU nodes - для ML, high-load, batch jobs, Spot / Preemptible nodes - дешевые, могуть быть удалены в любой момент)

# Kubernetes Deployment: netology-ml

## Описание
В рамках данного задания создан Kubernetes-манифест для развёртывания приложения
на базе **Apache Tomcat версии 8.5.69**.

Манифест описывает объект **Deployment** с использованием стратегии обновления
**RollingUpdate** и количеством реплик **2**.

RollingUpdate и replicas настраиваются через Deployment,
а Deployment не может иметь apiVersion: v1 — корректная версия apps/v1

---

## Параметры Deployment
- **Название:** netology-ml  
- **Тип ресурса:** Deployment  
- **Количество реплик:** 2  
- **Стратегия обновления:** RollingUpdate  
- **Контейнер:** tomcat:8.5.69  
- **Порт контейнера:** 8080
